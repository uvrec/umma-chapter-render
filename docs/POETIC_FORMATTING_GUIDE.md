# Керівництво з форматування поетичних текстів

## Проблема

WYSIWYG-редактори (такі як Tiptap) автоматично перетворюють кожен рядок на параграф `<p>` або список `<li>`, що створює небажані візуальні відступи між рядками. Це особливо проблематично для:

- Підписів (наприклад, "ом̇ тат сат")
- Імен та дат
- Поетичних текстів з коротких рядків
- Будь-яких текстів, де потрібно зберегти переноси рядків без відступів

## Рішення

Використовуйте клас `poetic-block` для блоків тексту, де потрібно зберегти переноси рядків без автоматичних відступів.

### Як використовувати

#### 1. У HTML редакторі

Оберніть текст у `<div>` з класом `poetic-block`:

```html
<div class="poetic-block">
ом̇ тат сат
А.Ч. Бгактіведанта Свамі
14 березня 1968 року
День народження Господа Чайтаньї
Храм Шрі Шрі Радги-Крішни
Нью-Йорк, штат Нью-Йорк
</div>
```

#### 2. У Tiptap редакторі

1. Перейдіть у режим HTML (якщо доступний)
2. Додайте клас `poetic-block` до блокового елемента
3. Або створіть новий `<div>` з класом `poetic-block`

### Приклад "до" та "після"

#### ❌ До (з автоматичними параграфами):

```html
<p>ом̇ тат сат</p>
<p>А.Ч. Бгактіведанта Свамі</p>
<p>14 березня 1968 року</p>
<p>День народження Господа Чайтаньї</p>
<p>Храм Шрі Шрі Радги-Крішни</p>
<p>Нью-Йорк, штат Нью-Йорк</p>
```

**Результат:** Великі відступи між рядками

#### ✅ Після (з `poetic-block`):

```html
<div class="poetic-block">
ом̇ тат сат
А.Ч. Бгактіведанта Свамі
14 березня 1968 року
День народження Господа Чайтаньї
Храм Шрі Шрі Радги-Крішни
Нью-Йорк, штат Нью-Йорк
</div>
```

**Результат:** Рядки розташовані один під одним без зайвих відступів

## Що робить `poetic-block`?

CSS стилі для цього класу:

```css
.poetic-block {
  white-space: pre-line;  /* Зберігає ручні переноси рядків */
  display: block;
  line-height: 1.35;      /* Компактне міжряддя */
  margin: 0;
  padding: 0;
}
```

- `white-space: pre-line` — браузер зберігає ручні переноси рядків, але не показує зайві пробіли
- `display: block` — поводиться як нормальний текстовий блок, не список
- Ніяких `<p>`, `<li>`, `<br>` — редактор не додає зайвих тегів

## Якщо редактор все одно вставляє `<p>` або `<div>`

Не проблема! CSS автоматично прибирає відступи всередині `poetic-block`:

```html
<div class="poetic-block">
  <p>ом̇ тат сат</p>
  <p>А.Ч. Бгактіведанта Свамі</p>
  <p>14 березня 1968 року</p>
</div>
```

Стилі автоматично прибирають візуальні відступи завдяки правилу:

```css
.poetic-block p,
.poetic-block div {
  margin: 0 !important;
  padding: 0 !important;
  display: block;
  white-space: pre-line;
}
```

## Коли використовувати

✅ **Використовуйте** `poetic-block` для:
- Підписів і посвят
- Імен, дат, місць
- Коротких поетичних текстів
- Будь-яких текстів з переносами рядків без відступів

❌ **НЕ використовуйте** для:
- Звичайних параграфів з відступами
- Списків (`<ul>`, `<ol>`)
- Таблиць
- Довгих текстових блоків з форматуванням

## Сумісність

Клас `poetic-block` працює в:
- TiptapRenderer (передмови, вступи)
- Будь-яких блоках з класом `prose` або `prose-reader`
- Dark mode (автоматично адаптується)

## Приклади використання в проекті

### Передмова книги

```html
<div class="poetic-block">
ом̇ тат сат
А.Ч. Бгактіведанта Свамі
Прабгупада
</div>

<p>Основний текст передмови починається тут...</p>
```

### Дата та місце

```html
<div class="poetic-block">
14 березня 1968 року
День народження Господа Чайтаньї
Храм Шрі Шрі Радги-Крішни
Нью-Йорк, штат Нью-Йорк
</div>
```

## Питання та підтримка

Якщо у вас виникли проблеми з форматуванням:
1. Перевірте, чи правильно застосовано клас `poetic-block`
2. Переконайтеся, що текст знаходиться всередині одного блоку
3. У разі проблем зверніться до технічної команди
